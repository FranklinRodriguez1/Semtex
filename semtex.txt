Propósito y Objetivos Propósito: Crear un SaaS de automatización inteligente (Copiloto Administrativo y Financiero) que permita a las PyMEs semi-automatizar tareas repetitivas de análisis de datos y comunicación mediante lenguaje natural, garantizando la privacidad absoluta de sus registros contables. Objetivo General: Desarrollar un agente autónomo de IA capaz de interpretar archivos estructurados (Excel) y ejecutar acciones en el entorno real (envío de correos estructurados) bajo una arquitectura multi-inquilino segura. • Procesar y estructurar datos dinámicos de hojas de cálculo desde un ecosistema Java sin perder consistencia relacional. • Implementar Function Calling orquestado desde la JVM para interactuar dinámicamente con el modelo de IA y activar servicios. • Validar el uso de modelos de lenguaje de código abierto locales (LLMs) para asegurar el 100% de confidencialidad. 2. Funciones Clave del MVP • Ingesta: Carga de archivos Excel (.xlsx, .xls) y .csv mediante una interfaz drag-and-drop administrada en el cliente. • Análisis: Consultas en lenguaje natural sobre balances, ingresos, egresos y comparativas financieras temporales. • Automatización: Redacción y envío automático de correos corporativos utilizando servicios de mensajería desde el backend. • Agente IA: Orquestación mediante Function Calling para extraer parámetros del chat y ejecutar acciones de forma autónoma. • Seguridad SaaS: Aislamiento multi-inquilino estricto por empresa con control de acceso basado en roles (RBAC). • Auditoría: Registro histórico de conversaciones y logs detallados de las acciones tomadas por el agente en la base de datos. 3. Inputs del Sistema (Requerimientos de Entrada) • Archivos de Datos: Documentos contables y financieros físicos subidos por el usuario en el módulo de carga. • Prompts de Usuario: Mensajes e instrucciones de texto ingresados en la interfaz de chat (ej. '¿Cuál es la ganancia de mayo?'). 6. Autenticación y Control de Accesos (RBAC) • Autenticación: Gestión de identidad delegada en Supabase Auth con validación y desencriptación de tokens JWT en el filtro de seguridad de Java. • Administrador (Admin): Control total de usuarios, APIs de correo, eliminación de archivos y lectura/escritura general. • Operador (Auxiliar): Carga de Excels, consultas al chat con Semtex y ordenamiento de envío de correos.
